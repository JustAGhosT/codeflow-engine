## Changelog Format Guidelines

### Categories

- **Added:** New features
- **Changed:** Changes in existing functionality
- **Deprecated:** Soon-to-be removed features
- **Removed:** Removed features
- **Fixed:** Bug fixes
- **Security:** Security fixes

### Version Format

- Use [Semantic Versioning](https://semver.org/)
- Format: `[MAJOR.MINOR.PATCH] - YYYY-MM-DD`
- Link to release: `[1.2.0]: https://github.com/org/repo/releases/tag/v1.2.0`

### Entry Format

- Use present tense ("Add feature" not "Added feature")
- Reference issues/PRs: "Fix bug (#123)"
- Group related changes
- Be specific and clear

### Examples

```markdown
### Added
- New GitHub App installation flow
- Support for Linear API v2
- Health check endpoint at `/health`

### Changed
- Improved error handling in PR processing
- Updated to Python 3.12
- Enhanced logging format

### Fixed
- Fixed issue with webhook processing (#456)
- Fixed memory leak in workflow execution (#789)
- Fixed authentication token expiration

### Security
- Fixed XSS vulnerability in dashboard
- Updated dependencies with security patches
```
